# 포디엄 (프로토타입)

클래식 음악인을 위한 플랫폼 프로토타입 — 뮬(Mule)의 클래식 버전을 목표로.
**범위 = 서양 클래식**: 성악·기악·지휘·작곡·이론(대위법·화성법 등). 국악은 자매 사이트 풍류로, 실용음악·무용은 제외.

## 페이지

- **구인구직** (`jobs.html`) — 공식 채용공고(크롤러 자동 수집) + 개인·팀 소규모 글 통합 보드
- **연습실** (`practice.html`) / **과외** (`lessons.html`) / **중고거래** (`market.html`) — 목데이터 프로토타입
- **커뮤니티** (`community.html`) — 게시판 초안 (예시글 + 로컬 글쓰기/댓글)

## 태그 체계 (단일 축: "연주냐, 가르치냐. 가르치면 누구를.")

| 태그 | 내용 |
|---|---|
| 연주 | 단원·객원·반주·지휘 + 오브리(교회·행사 — 별도 태그 아닌 하위 필터) |
| 교육 — 대학 | 교수·시간강사·겸임 (음대생 대상) |
| 교육 — 입시·전공 | 예중·예고 실기강사, 입시 레슨 |
| 교육 — 취미·입문 | 학원·문화센터·방과후·일반학교 (음악교사 포함) |
| 미분류 | 규칙에 안 걸림 — 추측하지 않고 사람이 확인 (크롤 로그에 출력) |

자격요건은 태그가 아니라 **필드**: `certReq`(교원자격증 예/아니오/무관) · `degreeReq`(학위) · `careerReq`(경력 무관/필요/미기재). 연령(`ageGroup`)은 필터 없이 배지 전용(현재 플래그 off).

## 크롤러

```
python crawler/main.py          # 폴링 주기에 맞는 소스만
python crawler/main.py --all    # 전 소스 강제 (hibrain 포함) — 분류기 수정 후 검증용
```

파이프라인: 소스 파서(`sources.py`, 교향악단·교육청·hibrain·아트인포 등) → 제목 필터(`relevant`/`musician_relevant`) → dedup(기관|악기, 악기추가 재공고 병합) → enrich(마감일·본문 + **첨부 HWP/HWPX/XLSX/PDF/OCR 추출**로 대학 전공·교과목 확인, 비음악 대학 제외) → 분류 재적용 → `data/official.json`·`official-data.js` 내보내기.

- hibrain은 로그인 세션 필요(`hibrain_auth.py`), 대학 강사 초빙은 첨부 교과목표에서 음악 전공만 추출
- 매일 03:00 스케줄 실행 + 장애 시 텔레그램 알림
- `EXT_VER`(main.py): 추출기·분류기 버전. **올리면 이전 수집분 승계가 무효화되어 재추출됨**
