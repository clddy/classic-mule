# 포디엄 v2.0

클래식 음악인을 위한 **공고 집약 플랫폼**. 뮬의 클래식 버전이 아니라, 공개된 정보를 가장 완전하게 모으는 수직 스택.
**범위 = 서양 클래식**: 성악·기악·지휘·작곡·이론. 국악은 자매 사이트 풍류로, 실용음악·무용은 제외.

## 전략 (2026-07 재정립)

시장은 둘로 갈린다 — **신뢰 시장**(프리랜서 객원·대타·하루대타: 지인 기반 자정, 공략 불가 확정)과
**공고 시장**(공식 채용: 기관은 법적으로, 교회는 네트워크가 바닥나면 공고를 냄). 포디엄은 후자만 한다.

- **Layer 1 — 공고 집약** (코어): 원천 직접 파싱(HWP·XLSX·OCR), 음악 특화 필드(악기 제공·리허설 횟수),
  음악인 무료. 비치헤드 = **교회 상시 포지션**(반주·지휘·솔리스트).
- **Layer 2 — 공간**: 공공 연습 공간 집약(수수료 모델이 구조적으로 못 들어오는 영역). 민간은 외부 예약처 링크만.
- **Layer 3 — 디렉토리** (로드맵): 소개할 때 첨부하는 음악인 프로필 링크. 지금은 자리만 비워둠.

버린 것: 구직 게시판, 급구·긴급 매칭, 과외·중고거래·커뮤니티 섹션, 결제·환불 레이어, '지원하기'류 거래 기능.
판단 기준: *지인 소개(신뢰)를 대체하려 하는가 → 제거 / 공개 정보를 모아 보여주는가 → 유지·강화.*

## 페이지

- **홈** (`index.html`) — 공고 피드 (최신 공고 + 필터 진입)
- **구인구직** (`jobs.html`) — 공식 수집 공고 + 직접 등록 공고. 원천 기관명 태그
- **연습실** (`practice.html`) — 신청제·공공 집약 + 민간 외부 예약 링크
- **소개** (`about.html`) / **수집 원천** (`sources.html`) — 원천 기관 전체 목록·수집 시각
- lessons/market/community.html — 종료, index로 리다이렉트

## 공고 태그 체계

연주(단원·객원·행사·반주) / 교회(반주·지휘·솔리스트 — 상시·정기 전용) / 교육 — 대학 / 교육 — 입시·전공 / 교육 — 취미·입문.
자격은 태그가 아니라 필드(certReq·degreeReq·careerReq). 미분류는 사람 확인 큐.

## 크롤러

```
python crawler/main.py          # 폴링 주기에 맞는 소스만 (스케줄: 매일 18시)
python crawler/main.py --all    # 전 소스 강제 — 분류기 수정 후 검증용
```

파이프라인: 소스 파서 → 음악인 필터 → dedup → enrich(마감·첨부 HWP/XLSX/OCR로 전공·교과목 추출)
→ 분류 재적용 → overrides(사람이 확인한 사실) 병합 → export. `EXT_VER` 올리면 승계 무효화·재추출.
헬스체크(`health_check.py`)가 소스별 수집량 baseline을 감시하고 이상 시 텔레그램 알림.
